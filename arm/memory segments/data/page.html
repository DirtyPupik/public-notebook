the .data section is used to store data - R+W
there's also .bss 

the .data section usually comes after .global and _start (but not necessarly).

The .data section is used to declare the memory region, where data elements are stored for the program. This section cannot be expanded after the data elements are declared, and it remains static throughout the program.

The .bss section is also a static memory section that contains buffers for data to be declared later in the program. This buffer memory is zero-filled.

inside .data, we put a label which functions as the variable name, a size type of the variable, and the contents
example: "list" is the label, ".word" is the size - 32bit signed int


ldr loads the address of "list" array starting point
