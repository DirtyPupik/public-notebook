Why use Linked List?
  • Regular arrays are fixed size, once declared, size cannot be changed during execution.
  • Dynamic memory allocation solved this problem with realloc(). yet as the array must stay continuous, when reallocating big chunks of memory, the whole array might need to be moved to another location, moving might slow things down.
  • also, adding an element in the middle of an array will cause all elements above to be shifted - lots of copying and things will also slow down alot
  • deleting an element from a middle will also cause a shift (copy).


In a linked list, each element stand's alone as a node. in a memory point of view, each element gets a memory allocation and a pointer that points to the next element on the list. nodes can be anywhere in memory.
Thus it is clear how easy can deletion and insertion of the middle may be done, it is only required to change 2 pointers.
Node's in C are mostly structs.

linked list termination:
  • node is a list item, it has a pointer and an element
  • edge pointers / edges - they connect the nodes
  • head - the first node in the list, used for keeping track.
  • tail - sometimes also used.
