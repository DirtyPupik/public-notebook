It is a type of initialized read only string pointer.
The pointer holds the address of the first element of the array.

Example:
char *amessage = "This is a string literal.";

They cannot be changed:
char* s = "hallo";
s[0] = 'W'; /* undefined behaviour */

So its better to make them constant, to cause compiler error:
char const* s1 = "welcome";
s1[0] = 'W'; /* compiler error! */


see also:
string literals in memory are stored in.rodata, and the linker dumps them in a read execute section (no write), see second answer.
