the zones are hirarchial
root servers are responsible for the root zone
TLD servers are responsible for the TLD zone

zones don't overlap. For example, the administrative authority of the TLD name server for the.com TLD doesn't encompass the google.com domain. Instead, it ends at the authoritative server responsible for google.com

Zone files- zones are configured through what are known as zone files, simple configuration files that declare all resource records for a particular zone.
zone files must contain SOA, NS records
SOA recorddeclares the zone and the name of the name server that is authoritative for it.
NS records indicate other name servers that may also be responsible for this zone.

reverse lookup zone filesthese let DNS resolvers ask for an IP, and get the FQDN associated with it returned.
these files contain pointer resource record declarations (a different type of record like A and AAAA). a PTR, or Pointer Record, resolves an IP to a name.
