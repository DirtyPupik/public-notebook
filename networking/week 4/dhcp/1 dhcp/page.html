DHCP is application layer but its purpose is to help configure the network layer

DHCP listens on UDP port 67

each computer on a TCP/IP network needs to have 4 things configured:
subnet mask
default gateway
name server
ip address

the first 3 are the same for every node on a network
but IP needs to be different for every single node

here comes DHCP to the rescue
Dynamic Host Configuration Protocol
DHCP is an application layer protocol that automates the configuration process of hosts on a network.

dynamic allocation- a range of IP addresses is set aside for client devices and one of these IPs is issued to these devices when they request one.
Under a dynamic allocation the IP of a computer could be different almost every time it connects to the network

automatic allocation - a range of IPs is set aside for assignment purpose

fixed allocation - requires manually input of MAC address to IP
its a sequrity measure to ensure communication to only certian devices
as such, see MAC spoofing

DHCP can also be used to assign NTP servers
NTP server - network time protocol - used to keep all computers on a network synchronized in time
