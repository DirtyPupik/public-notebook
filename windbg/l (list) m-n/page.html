• ln (List Nearest Symbols):
    • Syntax: ln Address
    • Purpose: The ln command is used to find the nearest symbols to a given memory address. It tries to determine the function name or label associated with the specified address and displays the result.
  • lm (List Loaded Modules):
    • Syntax: lm [Options]
    • Purpose: The lm command is used to list all the loaded modules (DLLs and executables) in the current debugging session. It provides information about the base addresses, sizes, and names of the loaded modules.

lm (List Loaded Modules) (userspace & kernel)


lm u list modules userspace only
lm k list modules kernel only
lm sm List Modules Sort Modules (Alphabetically)
lm f List Modules with Filesystem Path
lm v List Modules Verbose Output
