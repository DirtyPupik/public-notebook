• Module:
    • In the context of debugging, a module refers to a specific binary file (executable or DLL) that is loaded into the process's memory space.
    • Each module typically contains compiled code and data. When an application is executed, the operating system loads various modules into memory to run the program.
    • In Windbg, the lm command is used to list the loaded modules, displaying information such as base address, size, and name of each module.
  • Symbol:
    • A symbol is a unique identifier used by a debugger to represent elements in the code, such as functions, variables, and data structures.
    • Symbols provide human-readable names to memory addresses, making it easier for developers to interpret and debug code.
    • Symbol files (PDBs for Windows binaries) contain the mapping between symbol names and their corresponding memory addresses. These files are generated during the compilation process and are essential for meaningful debugging.
    • The ln command in Windbg allows you to find the nearest symbols to a given memory address, providing information about the function name or label associated with that address.
