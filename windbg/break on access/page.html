ba - break on access (processor / hardware breakpoints)

ba use's hardware breakpoints (4 debug registers).

The types of ba are:
w write
r read+write
e execute size must be 1 if the type is "e"
i i/o access

***Hardware breakpoints must be set after execution have started, because hardware debug registers are saved and restored as part of thread state, when starting execution, breakpoint set before are cleared momentarly.

A simplified form of the break on access command is:

ba <access type> <data size> <address>

Where:
  • <access type> == w (for write-access)
  • <data size> == 1, 2, 4, or 8 (bytes)
  • <address> == an absolute address, symbol name, or calculation of an address like "rsp+0x24"

So for instance to break when 4 bytesare writtento address 00000000`0014fdf4, you would enter:

  • ba w 4 00000000`0014fdf4

There is no break on read-only. there is however break on read and write
where w changes to r:
  • ba r 4 00000000`0014fdf4
