Many GDB commands support a format specifier, which we will call /FMT (because that's what it's called in the gdb help command for each of these commands)

/FMT is the combination of the /character,
followed by a number n (item count),
followed by a format specifier f,
and a unit size specifier u.
Giving /nfuformat.
Each of n, f, and u are optional, and will default to the last-used values or default initial values if left unspecified.
/10xw - show 10 items, in hexadecimal form, each one 4 byte long.


Format specifiers:
s - string (presumed null terminated)
i - instruction
d - decimal (signed)
u - unsigned decimal
f - float
x - hexadecimal
c - ASCII char
o - octal (binary representation)
t - binary ("t" stands for "two")
a - address, both absolute in hexadecimal and as an offset from the nearest preceding symbol.
z - like "x" but with padding of leading zero's to the size of the integer type.
r - raw
Documentation


Sizes (u):
*NOTE - GDB word size is different than intel's (intel word is 2 bytes).
  • b for bytes
  • h for half-words (2 bytes)
  • w for words (4 bytes)
  • g for giant-words (8 bytes)
