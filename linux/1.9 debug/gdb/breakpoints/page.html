Breakpoints (software):
  • Implemented as "interrupt 3" (0xCC) that the debugger is inserting into the code
  • Allows break over instruction execution only.
  • Can be done multiple number of times (unlimited number of bp).

Setting:
  • (b / break) main / *address / *main + 8
Listing:
  • info breakpoints / info break / info b
Clear (by address or function name):
  • clear *0xADDRESS / function_name
Delete (by breakpoint number):
  • d / delete b_number
Disabling (by breakpoint number):
  • disable b_number
Enabling  (by breakpoint number):
  • enable b_number


Hardware Break On Execute - implemented by the MMU
  • Limited to 4 Debug Registers (DR) on x86.
  • Allows breaking on read or read/write and execute (due to the limited number, its more common to use software break on execute).
  • Can be used to halt on non-execution accesses to memory locations.
  • hbreak / hb *adress / function_name
