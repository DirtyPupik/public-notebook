Char and block are considered special files (aka device files).
  • Special files provide an interface for devices in the form of a file in the file system.
  • They store content on the system
  • Read and write using special files happens immediately.
  • Char file acts as an interface to a char device - I/O happens one byte at a time (e.g. keyboard).
  • Block file atcs as an interface to a device which performs data I/O in units of blocks (e.g. physical storage device, ISO image file), size of blocks varies.

To test for a char or block device, use testor stat:

test -b FILE
FILE exists and is block special
test -c FILE
FILE exists and is character special

stat FILE
