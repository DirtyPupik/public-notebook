local variables lead to space allocation on the stack

there is a 16 byte alligment padding in the stack, "the stack will always be maintained 16 byte aligned"

local variables do not need to be stored on the stack in the same order they are defined in the higher level language
while fields in struct mush be stored the same order they are defined in the higher lever lan

usually in 32bit call conventions:
function parameters tend to get referenced as ebp+offset
local variables tend to get referenced as ebp-offset

When there's no return from main(), the compiler will automatically return 0 upon success.

  • argc into rdi
  • argv** into rsi
