RET - return from procedure

two forms
  • "ret" - most common, pop the top of the stack into RIP - pop increments RSP
  • (top of stack (RSP) -> RIP && RSP+0x08)
  • "ret 0x08" - less common, do the same as above, but also add a number of bytes to RSP
  • eg. ret 0x08 - add 8 bytes extra to RSP, ret 0x20 - add 32 bytes extra to RSP,
