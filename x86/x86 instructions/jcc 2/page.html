• Jump by condition. comes in many forms. will usually come after cmp instruction.
  • When using signed or unsigend notation, the compiler emits different instructions, this helps to understand the code written by the programmer and reverse engineer it.
  • From the Hardware prespective, it doesn't matter if the integer was signed or unsigned, the arithmithic operation will be done the in same manner

"
The hardware keeps various status flags in RFLAGs. Some are relevant to operations on signed numbers, some are relevant to ops on unsigned numbers. Some (e.g. like the zero flag) don't care whether it's signed or unsigned. The hardware sets all flags, based on internal logic which accounts for whether the values could have been signed or could have been unsigned. The compiler is what ultimately is responsible for emitting the correct assembly instructions which behind the scenes check the right bits to honor the behavior as if something had been been signed or unsigned."

  • JZ / JE (jump if equal, jump if zero): if ZF == 1
  • JNZ / JNE (jump if not equal, jump if not zero): if ZF == 0
  • JLE / JNG (jump if less than or equal, jump if not greater than): if ZF == 1 OR if SF != OF
  • JGE / JNL (jump if greater than or equal, jump if not less than): if SF == OF
  • JAE (jump if above or equal - unsigned notation)
  • Jump Above (ja) == Jump Not Below or Equal (jnbe)
  • Jump Not Above (jna) == Jump Below or Equal (jbe)
  • Jump Below (jb) == Jump Not Above or Equal (jnae)
  • Jump Not Below (jnb) == Jump Above or Equal (jae)
  • Jump Greater-than (jg) == Jump Not Less-than or Equal (jnle)
  • Jump Not Greater-than (jng) == Jump Less-than or Equal (jle)
  • Jump Less-than (jl) == Jump Not Greater-than or Equal (jnge)
  • Jump Not Less-than (jnl) == Jump Greater-than or Equal (jge)
