CALL's job is to transfer control to a different function, in a way that the control can be later resumed where it was left off.

first it pushes the address of the next instruction onto the stack (to be used by RET when procedure is done)
(PUSH RSP-8h, RIP)

than is changes RIP (instruction poiner) to the address given in the instructions

the destination address for the target function (the function being called) can be specified in multiple ways:
  • absolute address
  • relative address (relative to the end of instruction or some other register
